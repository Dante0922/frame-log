# frame-log 통합 서비스 계획서 (공통점 + 강점 반영)


## 1. 서비스 정의
- 서비스명: `frame-log`
- 한 줄 정의: 사진가를 위한 주간 출사지 추천 및 익명 리뷰 기반 탐색 서비스
- 타겟: 모바일 사용자 우선, 이후 PC 반응형 확장
- 핵심 가치:
  - 이번 주 출사지 선택 시간을 줄이는 빠른 큐레이션
  - 회원가입 없는 즉시 탐색/리뷰 참여

## 2. 핵심 UX 원칙
- 빠른 진입: 메인에서 금주의 추천 출사지를 즉시 노출
- 직관적 탐색: 카드/이미지 탭 한 번으로 상세 진입
- 즉시 의사결정: 상세에서 정보, 사진, 리뷰를 한 흐름으로 확인
- 낮은 참여 장벽: 익명 랜덤 닉네임으로 리뷰 작성

## 3. 사용자 플로우
```text
[앱 진입]
   ↓
[홈: 금주의 추천]
   ↓ (카드/이미지 탭)
[출사지 상세: 정보 + 사진 + 리뷰]
   ↔
[출사지 목록: 전체 탐색]
```

## 4. 화면 구성 (모바일 MVP)

### 4.1 홈
- 상단: 서비스 타이틀/헤더
- 본문: 금주의 추천 출사지 히어로 카드
- 액션: 카드 탭 시 상세 이동
- 내비게이션: 홈 / 출사지 목록

### 4.2 출사지 목록
- 카드 리스트: 대표 이미지, 장소명, 요약 정보
- 상호작용: 카드 탭 시 상세 이동

### 4.3 출사지 상세
- 출사지 기본 정보: 이름, 위치(주소/지역), 설명, 추천 촬영 시간대/계절
- 관련 사진: 갤러리 조회
- 리뷰: 목록 조회 + 동일 화면 내 작성

### 4.4 리뷰 작성
- 인증 없음, 익명 닉네임 자동 적용
- 짧은 텍스트 리뷰 작성 후 등록

## 5. MVP 기능 정의

| ID | 기능 | 설명 | 우선순위 |
|---|---|---|---|
| FR-01 | 금주의 추천 출사지 노출 | 홈에서 주간 1개 추천 출사지 노출 | P0 |
| FR-02 | 추천 카드 상세 이동 | 홈 카드/이미지 탭 시 상세 이동 | P0 |
| FR-03 | 출사지 상세 조회 | 기본 정보 + 관련 사진 + 리뷰 조회 | P0 |
| FR-04 | 출사지 목록 조회 | 전체 출사지 리스트 조회 및 상세 이동 | P0 |
| FR-05 | 익명 리뷰 작성 | 회원가입 없이 리뷰 작성 | P0 |
| FR-06 | 랜덤 닉네임 부여 | 예: `새찍는사진사777`, 기기 내 유지 | P0 |

## 6. 익명 리뷰 정책 (통합안)
- 로그인/회원가입 없음
- 최초 진입 시 랜덤 닉네임 생성
- 닉네임은 브라우저 로컬 저장소에 저장하여 동일 기기에서 유지
- 리뷰 저장 단위: `spot_id`, `nickname`, `content`, `created_at`
- 리뷰 길이 제한: 짧은 형식(초기 100자 내외)

## 7. 정보 구조 (IA)
- 홈
- 출사지 목록
- 출사지 상세
- 리뷰 조회/작성

## 8. 데이터 모델 초안

### 8.1 `spot`
- id, name, description, region, address, recommended_time, recommended_season, hero_image_url, is_weekly_featured, created_at

### 8.2 `spot_photo`
- id, spot_id, image_url, caption, sort_order, created_at

### 8.3 `spot_review`
- id, spot_id, nickname, content, created_at

## 9. API 초안 (MVP 범위)

| Method | Path | 설명 |
|---|---|---|
| GET | `/api/spots/weekly` | 금주의 추천 출사지 조회 |
| GET | `/api/spots` | 출사지 목록 조회 |
| GET | `/api/spots/{id}` | 출사지 상세 조회 |
| GET | `/api/spots/{id}/photos` | 출사지 사진 목록 조회 |
| GET | `/api/spots/{id}/reviews` | 출사지 리뷰 목록 조회 |
| POST | `/api/spots/{id}/reviews` | 익명 리뷰 작성 |

## 10. 기술 스택
- Frontend: React + TypeScript
- Backend: Kotlin + Spring Boot
- Database: MySQL
